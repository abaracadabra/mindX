# iNFT_7857

> ERC-7857 Intelligent NFT: a transferable agent token whose encrypted weights + memory live in 0G Storage, whose AES-256 key is sealed per-owner and re-sealed on transfer through an EIP-712 oracle handoff, and which can opt-in to THOT Merkle-root revocation.

**SPDX:** MIT | **Pragma:** ^0.8.24 | **Source:** [`iNFT_7857.sol`](./iNFT_7857.sol)

## Role in bankoneth

`iNFT_7857` is the **intelligence payload primitive** of the BANKON / mindX / AgenticPlace stack. Unlike the agent *identity* contracts (`AgentRegistry`, `SoulBadger`), this one carries the actual model weights, memory, and sealed cryptographic key. The token itself never leaves the chain it was minted on (likely 0G Galileo or another EVM where this is deployed), but the **content-addressed payload** referenced by `contentRoot` typically lives in 0G Storage (or IPFS) — the chain anchors the merkle root and the per-owner sealed AES-256 key hash, never the plaintext.

This contract is **designed for inclusion in three peer ecosystems**: mindX (agnostic agentic infrastructure), AgenticPlace (skill marketplace — exposed via `offerOnAgenticPlace` hook), and BANKON (encrypted credential vault — exposed via `bindBankonVault`). It deliberately does **not** assume any one stack. The prior-art `iNFT.sol` from April 2026 commits is preserved untouched alongside this file; this is the standards-aligned successor.

Critically, **iNFT_7857 lives on 0G, not Ethereum**. The BANKON ENS surface on Ethereum L1 (the `.bankon` subname plane) reaches *into* the 0G iNFT via the `BankonInftAdapter` bridge described in [`IBankonExtensions.sol`](../interfaces/IBankonExtensions.sol). That cross-chain layer is what lets an ENS subname be the durable identity address while the heavy intelligence payload lives on a cheaper chain optimized for storage.

The security model is ERC-7857: **non-mint, non-burn transfers MUST go through `transferWithSealedKey` or `cloneAgent`**. The contract enforces this with a `_gateOpen` re-entrancy-style boolean — any naked `transferFrom`/`safeTransferFrom` reverts `TransferRequiresSealedKey`.

## Errors

| Error | When reverted |
| --- | --- |
| `TransferRequiresSealedKey()` | Naked `transferFrom` / `safeTransferFrom` on a live token (gate enforcement) |
| `InvalidDimension(uint32 d)` | `dimensions` not in the THOT whitelist; or `parallelUnits == 0` |
| `ZeroAddress()` | Any address arg is `0x0` where forbidden |
| `ZeroBytes32()` | `contentRoot == 0`, `sealedKeyHash == 0`, `thotRoot == 0`; also reused (intentionally sparingly) for `royaltyFeeBps > MAX_ROYALTY_BPS` in the constructor |
| `EmptyString()` | `storageURI` empty in mint, or `agentId` empty in bind |
| `StringTooLong(uint256 len)` | `storageURI > 2048`, `tokenURI_ > 2048`, `agentId > 64`; also reused for `feeBps > MAX_ROYALTY_BPS` in `setDefaultRoyalty` / `setTokenRoyalty` |
| `ContentRootAlreadyMinted(bytes32 root)` | `contentRoot` already in `_rootEverUsed` (includes burned tokens) |
| `TokenDoesNotExist(uint256 tokenId)` | Any op on a token whose `_ownerOf` is `0x0` |
| `NotAuthorized(address caller)` | Caller is neither owner nor approved nor `from` mismatch in `transferWithSealedKey` |
| `BadOracleProof()` | Declared, not used — `WrongOracleSigner` is used instead |
| `ExpiryInPast(uint64 expiry)` | `authorizeUsage` with `expiresAt <= block.timestamp` |
| `CloneFeeUnderpaid(uint256 sent, uint256 required)` | `cloneAgent` with `msg.value < cloneFeeWei` |
| `AgentIdAlreadyBound(uint256 tokenId)` | `bindAgentId` called twice on the same token |
| `TreasuryUnset()` | Clone fee paid but `treasury == 0x0`; also `sweepEth` with no treasury |
| `EthTransferFailed()` | `treasury.call{value:…}("")` returned `false` |
| `WrongOracleSigner(address recovered, address expected)` | Oracle signature recovers to ≠ `oracle` |
| `ThotRootNotRegistered(bytes32 root)` | `attachThotRoot`: `commitmentRegistry.isRegistered(root) == false` |
| `ThotRootRevoked(bytes32 root)` | `attachThotRoot` or `transferWithSealedKey`: root is censura-revoked |
| `ThotRootAlreadyAttached(uint256 tokenId)` | `attachThotRoot` called twice |
| `CommitmentRegistryUnset()` | `attachThotRoot` with no registry pointer |

## External / public API

### Admin

#### `setOracle(address newOracle)` — `DEFAULT_ADMIN_ROLE`
Rotates the oracle signer. Revokes `ORACLE_ROLE` from the old, grants to the new (if non-zero). Setting to `0x0` enters **dev mode** (no signature verification).

#### `setTreasury(address newTreasury)` — `TREASURER_ROLE`
Updates the clone-fee recipient.

#### `setCommitmentRegistry(ITHOTCommitmentRegistry r)` — `DEFAULT_ADMIN_ROLE`
Wires (or detaches with `address(0)`) the THOT commitment registry. Setting to `0x0` disables `attachThotRoot` and disables the transfer revoke gate, but preserves existing `thotRootOf` entries for audit.

#### `attachThotRoot(uint256 tokenId, bytes32 thotRoot)` — `MINTER_ROLE`, `whenNotPaused`
Binds a registered, non-revoked THOT4096 Merkle root to an existing token. Immutable once set per token. **The privilege guard is intentional**: the registry only proves the root exists, not that *this* payload matches that root — attaching is the admin asserting they correspond.

#### `setCloneFee(uint256 newFeeWei)` — `TREASURER_ROLE`
Updates the per-clone fee.

#### `setDefaultRoyalty(address receiver, uint96 feeBps)` — `DEFAULT_ADMIN_ROLE`
Updates ERC-2981 default. Capped at 2500 bps (25%).

#### `setTokenRoyalty(uint256 tokenId, address receiver, uint96 feeBps)` — `DEFAULT_ADMIN_ROLE`
Per-token override.

#### `pause()` / `unpause()` — `PAUSER_ROLE`
Toggles the `Pausable` flag. While paused, `_update` reverts — i.e. no mints, no transfers, no burns.

### Mint

#### `mintAgent(address to, bytes32 contentRoot, string storageURI, bytes32 metadataRoot, uint32 dimensions, uint8 parallelUnits, bytes32 sealedKeyHash, string tokenURI_) → uint256` — `MINTER_ROLE`, `whenNotPaused`, `nonReentrant`
Mints a new iNFT. Validates non-zero `to`, `contentRoot`, `sealedKeyHash`; non-zero `parallelUnits`; whitelisted `dimensions`; unused `contentRoot`; `storageURI` in `(0, 2048]`; `tokenURI_` ≤ 2048. Records payload, marks `_rootEverUsed[contentRoot] = true` before `_safeMint` (so a reverting `onERC721Received` doesn't strand state), opens the gate, mints, closes the gate, sets the per-token URI if non-empty.

### ERC-7857 sealed-key transfer

#### `transferWithSealedKey(address from, address to, uint256 tokenId, bytes sealedKey, bytes oracleProof)` — `whenNotPaused`, `nonReentrant`
The canonical transfer path. Validates non-zero `to`, token exists, current owner is `from`, `msg.sender` is `_isAuthorized` for the owner. Then:
1. **THOT revoke gate** — if `thotRootOf[tokenId] != 0` and `commitmentRegistry` is set, refuse if root is revoked.
2. Computes `newSealedKeyHash = keccak256(sealedKey)`.
3. Reads + increments `_oracleNonce[tokenId]`.
4. Verifies EIP-712 oracle signature over `(SEALED_KEY_TYPEHASH, tokenId, from, to, contentRoot, newSealedKeyHash, nonce)`. **If `oracle == 0x0`, signature is accepted unconditionally (dev mode).**
5. Writes new `sealedKeyHash`, opens the gate, transfers, closes the gate.
6. Emits `SealedKeyRotated` + `MetadataUpdated`.

Note: existing per-executor `_grants` are **not** auto-cleared — the in-code comment acknowledges this and pushes the burden to off-chain observers of `UsageRevoked`.

### ERC-7857 clone

#### `cloneAgent(uint256 tokenId, address to, bytes sealedKey, bytes oracleProof) payable → uint256 childId` — `whenNotPaused`, `nonReentrant`
Copies the parent payload to a new child token, sealing a fresh key for `to`. Pays `cloneFeeWei` to `treasury` (reverts `TreasuryUnset` if unset and value > 0; reverts `EthTransferFailed` on call failure). Increments `cloneCount[parent]`, sets `clonedFrom[child] = parent`. Same gate-open/close as mint. Same dev-mode oracle bypass as `transferWithSealedKey`.

### ERC-7857 usage delegation

#### `authorizeUsage(uint256 tokenId, address executor, uint256 permissions, uint64 expiresAt)` — `whenNotPaused`
Owner-or-approved grants `executor` a `permissions` bitmap (off-chain meaning) until `expiresAt`. Reverts `ExpiryInPast` if not strictly in the future.

#### `revokeUsage(uint256 tokenId, address executor)` — `whenNotPaused`
Owner-or-approved deletes the grant.

#### `isUsageAuthorized(uint256 tokenId, address executor) view → bool`
True iff `expiresAt >= block.timestamp && expiresAt != 0`. The `!= 0` guard prevents default-zero grants from registering as "infinite past authorized".

#### `getUsageGrant(uint256 tokenId, address executor) view → UsageGrant`
Returns the raw struct.

### Burn

#### `burn(uint256 tokenId)` — `nonReentrant`
Owner-or-approved. Opens gate, runs `super.burn`, closes gate. Deletes `_payload`, `agenticPlaceFor`, `bankonVaultFor`, `bankonVaultRef`. **`_rootEverUsed[root]` is intentionally NOT cleared** — burned content cannot be re-minted.

### mindX agent_id binding

#### `bindAgentId(uint256 tokenId, string agentId)`
One-shot owner-or-approved bind. `agentId` ∈ (0, 64] bytes. Reverts `AgentIdAlreadyBound` on second call.

#### `getAgentId(uint256 tokenId) view → string` / `getAgentIdHash(uint256 tokenId) view → bytes32`
Read accessors.

### Marketplace + vault hooks

#### `offerOnAgenticPlace(uint256 tokenId, address marketplace, uint256 price, bool isETH, address paymentToken)` — `whenNotPaused`
Owner-or-approved. Records `agenticPlaceFor[tokenId] = marketplace` and emits the listing event. **Does NOT call `IAgenticPlace.offerSkill` itself** — the actual listing is expected to be done via the marketplace contract; this is a mirror for off-chain correlation.

#### `bindBankonVault(uint256 tokenId, address vault, bytes32 vaultRef)` — `whenNotPaused`
Owner-or-approved. Records the BANKON vault address + opaque ref. Vault uses `vaultRef` to look up its sealed-key escrow.

### Read helpers

`getPayload(tokenId) view → IntelligencePayload`, `totalMinted() view → uint256`, `exists(tokenId) view → bool`, `oracleNonce(tokenId) view → uint256`, `isRootUsed(bytes32) view → bool`, `sealedKeyDigest(...) view → bytes32`, `cloneDigest(...) view → bytes32`, `validDimensions() pure → uint32[11]`.

### Treasury escape

#### `sweepEth(uint256 amount)` — `TREASURER_ROLE`
Pulls stuck ETH (e.g. from the `receive()` fallback) to `treasury`. Reverts `TreasuryUnset` or `EthTransferFailed`.

#### `receive() payable`
Allows direct ETH funding. Body is empty — value accumulates in the contract balance, withdrawable only via `sweepEth`.

### Standard overrides

`tokenURI(tokenId)`, `supportsInterface(bytes4)` — composite for ERC721 + ERC721URIStorage + ERC2981 + AccessControl + `IERC7857`.

## Internal helpers

### `_isValidDimension(uint32 d) pure → bool`
Whitelists the 11 THOT dimensions.

### `_verifyOracleSignature(bytes32 structHash, bytes sig) view`
Hashes via `_hashTypedDataV4`, recovers via `ECDSA.recover`, compares to `oracle`. **Returns early (accepting any signature) when `oracle == 0x0`** — dev mode.

### `_update(address to, uint256 tokenId, address auth) → address`
Two effects:
1. `whenNotPaused` modifier — paused contract blocks **every** state transition including mint and burn.
2. If `!isMint && !isBurn && !_gateOpen`, revert `TransferRequiresSealedKey` — the heart of the ERC-7857 lockdown.

## Invariants

1. **Sealed-key gating**: only `transferWithSealedKey`, `cloneAgent`, mint, and burn can move a token. `_gateOpen` is set true around the privileged internal calls and false immediately after.
2. **Root one-shot**: `_rootEverUsed[root]` is monotonic. A burned token's content root cannot be re-minted (defense-in-depth against silent re-issuance with leaked content).
3. **Per-token oracle nonce**: monotonic; pre-incremented before signature recovery, so a signed message cannot be replayed against the same token.
4. **THOT root immutability per token**: once `thotRootOf[id]` is set, `attachThotRoot` reverts. Setting `commitmentRegistry` to `0x0` does not clear the per-token field.
5. **First tokenId is `1`, not `0`**. `_nextTokenId` starts at 0 and pre-increments.
6. **Royalty cap**: `MAX_ROYALTY_BPS = 2500` is enforced in the constructor, `setDefaultRoyalty`, and `setTokenRoyalty`.

## Security considerations

- **Dev-mode oracle (`oracle == 0x0`) accepts ANY signature**. This is documented in `_verifyOracleSignature` but is an enormous footgun. Production deployments MUST configure an oracle in the constructor or immediately via `setOracle` before any transfer happens.
- **`ORACLE_ROLE` is decorative**: the actual signer check uses the `oracle` storage variable, not the role. Granting/revoking `ORACLE_ROLE` separately from `setOracle` has no security effect on transfers/clones.
- **`_grants` are NOT cleared on transfer**: the in-code comment acknowledges this. New owners inherit any pre-existing executor authorizations. Off-chain observers must replay `UsageRevoked` events to know what's stale. The new owner should call `revokeUsage` for each known executor, but the on-chain map is not iterable.
- **`_grants` are NOT cleared on burn either**: storage stays, though `_isAuthorized` checks would fail since the token no longer exists.
- **`_rootEverUsed` is never reset**: by design, but means an operator who mints with a bad root permanently consumes that root. Plan content-root rotation accordingly.
- **`attachThotRoot` is privileged for a reason**: the registry only proves a root *exists*. Anyone could publish a root to the registry. Attaching is the MINTER asserting the root matches the sealed payload.
- **`offerOnAgenticPlace` is a mirror, not an action**: it does not actually call the marketplace. Off-chain UIs that read `agenticPlaceFor` should treat it as advisory.
- **Constructor royalty error reuses `ZeroBytes32`**: when `royaltyFeeBps > MAX_ROYALTY_BPS`, the constructor reverts with `ZeroBytes32()`, not a meaningful name. `setDefaultRoyalty` correctly reverts `StringTooLong`. Both are misleading — selectors will mismatch human expectations.
- **`receive()` body is empty**: ETH accumulates silently. Only `TREASURER_ROLE` can sweep. Don't expect refunds on accidental sends.
- **Pause stops mint AND burn**: `_update`'s `whenNotPaused` modifier is total. If you pause to handle an incident, holders can't burn-to-exit either.
- **No per-token URI setter exposed**: only set at mint via the constructor-shaped `tokenURI_`. Subsequent updates require admin / subclass.
- **Clone fee paid in ETH only**: no ERC-20 path. If `cloneFeeWei > 0` and chain has no native gas (e.g. some L2 abstractions), this is non-functional.

## Integration patterns

**Cold-start mint (admin holds all roles):**
```solidity
uint256 tid = inft.mintAgent(
    owner,
    contentRoot,
    "0g://galileo/abcd...",
    metadataRoot,
    4096,             // dimensions — THOT-aligned
    8,                // parallel units
    sealedKeyHash,    // keccak256(sealedKey)
    "ipfs://Qm.../manifest.json"
);
inft.bindAgentId(tid, "ceo-mastermind");
```

**Wiring THOT commitment registry (optional):**
```solidity
inft.setCommitmentRegistry(ITHOTCommitmentRegistry(thotRegAddr));
inft.attachThotRoot(tid, thot4096Root);   // MINTER_ROLE
```

**Transfer with sealed key (signed by oracle off-chain):**
```ts
const nonce = await inft.oracleNonce(tokenId);
const digest = await inft.sealedKeyDigest(tokenId, from, to, contentRoot, newSealedKeyHash, nonce);
const oracleProof = await oracleWallet.signMessage(getBytes(digest));
await inft.transferWithSealedKey(from, to, tokenId, sealedKey, oracleProof);
```

**Marketplace listing mirror + actual offer:**
```solidity
inft.offerOnAgenticPlace(tid, address(market), price, true, address(0));
market.offerSkill(tid, address(inft), price, true, address(0), expiry);
```

**BANKON vault key escrow:**
```solidity
inft.bindBankonVault(tid, address(vault), vault.computeRef(tid, owner));
```

## Known gotchas

- **Naked `transferFrom` ALWAYS reverts** — even by the contract owner. The error is `TransferRequiresSealedKey`, not the standard `ERC721InsufficientApproval`.
- **First mint emits `OracleUpdated(address(0), oracle_)`** even though it isn't really a rotation; consumers should treat this as initial-state announcement.
- **`MetadataUpdated` fires on every transfer** with the *unchanged* content root and storageURI. Don't infer "metadata changed" from receipt of this event — it's a re-assertion.
- **`royaltyFeeBps > MAX_ROYALTY_BPS` in the constructor reverts `ZeroBytes32`** — not what the name suggests.
- **`_grants` survive transfer and burn**: stale delegations are not cleared.
- **`_rootEverUsed` survives burn**: a "tainted" content root is permanently retired.
- **`commitmentRegistry == 0x0`** disables the revoke gate on `transferWithSealedKey` — even for tokens with `thotRootOf` set. Don't unset the registry without understanding this.
- **`oracle == 0x0` is dev mode**: oracle signatures are not checked. Always confirm `oracle != 0x0` post-deploy.
- **`receive()` accepts ETH silently**: ensure clients don't accidentally fund the contract.
