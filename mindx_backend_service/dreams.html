machine.dreaming — consolidation, in the open

information → consolidation → knowledge → concept → wisdom → THOT

Every ~8h mindX runs a 7-phase dream cycle that takes raw STM (per-agent process traces, hours of accumulated data) and consolidates it into LTM (durable insights). The cycle distributes, it does not delete — pruned STM moves to archive; consolidated insights become LTM JSON; and a parallel *_training.jsonl file lands alongside, suitable for finetuning a wisdom-aware model. Over time, repeated dream-extracted patterns are verified against new outcomes; once a pattern crosses the verification threshold it becomes machine.wisdom — anchored on chain as a THOT.

This page shows what the dream cycle actually does: bytes saved, STM input vs LTM output side-by-side, and the training rows produced.

The system dreams. Every cycle, mindX looks back at the noise of recent operation — the process traces, the tool calls, the abandoned plans — and asks: what pattern repeated? What survived the friction of execution? Those survivors become insights. The insights become knowledge. The knowledge, verified across cycles, becomes wisdom. The wisdom, anchored on chain, becomes THOT — public, citable, transferable. The dream is not where information dies. It is where information learns to count.

training data — the wisdom substrate

why *_training.jsonl matters

Every dream cycle now writes a sibling *_training.jsonl file alongside the raw *_dream_insights.json. Each line is one fine-tuning example in the OpenAI/Anthropic chat-completion shape ({"messages": [{role, content}, …]}) — directly usable by openai api fine_tunes.create, anthropic-tools, or local LoRA training (Axolotl, Unsloth, vLLM-LoRA). The system message gives the consolidation persona; the user message is a real STM excerpt; the assistant message is the consolidated insight. As cycles accumulate, this corpus becomes a domain-specific instruction-tuning dataset — the substrate for promoting verified patterns to machine.wisdom and ultimately on-chain THOT mints. A later phase will compose the verified subset into a single curated jsonl, embed it through pgvector, and offer it as a public dataset (CID-anchored on ARC).
