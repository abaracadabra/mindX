| [`iNFT_7857.sol`](../daio/contracts/inft/iNFT_7857.sol) | **ERC-7857 + ERC-2981 + AccessControl + Pausable + EIP-712** | **Hardened production build** | Standards-aligned. The legacy `iNFT.sol` is reference + timestamp evidence; the production substrate is `iNFT_7857.sol`. Both retain the same 11-dimension THOT whitelist for off-chain compatibility.

## Design rationale (why each piece exists)

### 1. The ERC-7857 problem
A "regular" NFT can be transferred with a single `transferFrom` call — the new owner gets the token, the URI metadata, and that's it. For an **intelligent** NFT carrying encrypted weights / persona / memory, this is broken: the new owner inherits the token but cannot decrypt the intelligence. ERC-7857 addresses this by mandating that transfers re-seal the intelligence to the new owner.

### 2. Transfer gating (the critical fix)
The contract overrides `_update()` to **revert** any standard `transferFrom` / `safeTransferFrom`. Mints (from `address(0)`), burns (to `address(0)`), and gated calls (`transferWithSealedKey`, `cloneAgent`) pass; everything else hits `TransferRequiresSealedKey()`. This is enforced by 3 dedicated tests including the SafeTransfer-with-data variant.

```
   transferFrom / safeTransferFrom    →  REVERT  (TransferRequiresSealedKey)
   transferWithSealedKey              →  ok (gate opens, oracle verifies, key rotates)
   cloneAgent                         →  ok (gate opens, oracle verifies, child mints)
   mint  (from == address(0))         →  ok
   burn  (to == address(0))           →  ok
```

### 3. EIP-712 oracle proof
`transferWithSealedKey` and `cloneAgent` require an oracle ECDSA signature over a domain-separated typed-data digest:

```solidity
bytes32 SEALED_KEY_TYPEHASH = keccak256(
    "SealedKeyHandoff(uint256 tokenId,address from,address to,bytes32 contentRoot,bytes32 newSealedKeyHash,uint256 nonce)"
);
bytes32 CLONE_TYPEHASH = keccak256(
    "AgentClone(uint256 parentTokenId,address to,bytes32 contentRoot,bytes32 newSealedKeyHash,uint256 nonce)"
);
```

The per-token `oracleNonce` increments after every successful re-encryption, blocking replay. View helpers `sealedKeyDigest(...)` and `cloneDigest(...)` let off-chain signers compute the digest via JSON-RPC without rebuilding the EIP-712 envelope themselves.

### 4. AccessControl — four roles
`Ownable` is replaced with OpenZeppelin AccessControl so AgenticPlace marketplace, BANKON vault, and IDManagerAgent can each be granted minting rights independently:

| Role | Holder examples | What it gates |
|------|-----------------|---------------|
| `DEFAULT_ADMIN_ROLE` | mindX operator (codephreak) | grant/revoke roles, set oracle, set per-token royalty |
| `MINTER_ROLE` | AgenticPlace contract, BANKON vault, IDManagerAgent | `mintAgent` |
| `ORACLE_ROLE` | TEE / ZK verifier signer | passive marker; verification uses the `oracle` address directly |
| `PAUSER_ROLE` | mindX operator + on-call | `pause` / `unpause` (emergency stop on mints, transfers, clones, authorizations, bindings) |
| `TREASURER_ROLE` | DAIO Treasury | `setTreasury`, `setCloneFee`, `sweepEth` |

### 5. Per-(token, executor) usage grants
The legacy contract had a single grant slot per token — granting Bob silently destroyed Alice's grant. The new contract uses `mapping(tokenId => mapping(executor => UsageGrant))` plus an explicit `revokeUsage(tokenId, executor)` function. Each grant carries `permissions` (uint256 bitmap interpreted off-chain), `expiresAt` (uint64), and `grantor` (address that issued it).

### 6. Bindings — agent_id, AgenticPlace, BANKON
Three on-chain hooks tie iNFT-7857 into the rest of the mindX ecosystem:

- **`bindAgentId(tokenId, agentId)`** — one-shot binding to a mindX agent_id string (e.g. `ceo-mastermind`). Stored as both raw string and `keccak256` hash for off-chain indexer convenience. Reverts if already bound.
- **`offerOnAgenticPlace(tokenId, marketplace, price, isETH, paymentToken)`** — records a marketplace listing. The actual `offerSkill()` call goes to the AgenticPlace contract; this stores the pointer so off-chain readers (and the [/openagents](../openagents/README.md) dashboard) can correlate.
- **`bindBankonVault(tokenId, vault, vaultRef)`** — registers that the sealed AES key for this token is escrowed in a BANKON vault. `vaultRef` is opaque — the vault uses it to look up the key on owner-authenticated retrieval.

### 7. Burn with permanent root reservation
`burn(tokenId)` (from ERC721Burnable) clears the `IntelligencePayload` plus AP/BANKON bindings. **`_rootEverUsed[contentRoot]` stays `true`** — the same root can never be silently re-minted. This blocks an attack where a leaked content payload is repackaged as a "new" iNFT after the original was burned. Tested.

### 8. Input validation
All write paths validate inputs with named errors (cheaper revert reasons than strings):

| Error | When it fires |
|-------|---------------|
| `ZeroAddress()` | Recipient, oracle, marketplace, vault, royalty receiver = 0x0 |
| `ZeroBytes32()` | `contentRoot` or `sealedKeyHash` = bytes32(0) |
| `EmptyString()` | empty `storageURI` or `agentId` |
| `StringTooLong(uint256)` | URI > 2048 bytes, agent_id > 64 bytes, royalty > 2500 bps |
| `InvalidDimension(uint32)` | dimension not in the 11-element whitelist; also fires on `parallelUnits == 0` |
| `ContentRootAlreadyMinted(bytes32)` | content root previously used (even if its token was burned) |
| `TokenDoesNotExist(uint256)` | tokenId never minted or already burned |
| `NotAuthorized(address)` | caller is neither owner, approved-for-token, nor approved-for-all |
| `ExpiryInPast(uint64)` | `authorizeUsage` with `expiresAt <= block.timestamp` |
| `BadOracleProof()` / `WrongOracleSigner(address recovered, address expected)` | EIP-712 signature does not recover to the configured oracle |
| `CloneFeeUnderpaid(uint256 sent, uint256 required)` | `cloneAgent` `msg.value < cloneFeeWei` |
| `AgentIdAlreadyBound(uint256)` | second `bindAgentId` call on the same token |
| `TreasuryUnset()` / `EthTransferFailed()` | clone-fee routing failures |
| `TransferRequiresSealedKey()` | standard `transferFrom` / `safeTransferFrom` attempted |

## Function reference

### Mint

| Function | Role | Effect |
|----------|------|--------|
| `mintAgent(to, contentRoot, storageURI, metadataRoot, dimensions, parallelUnits, sealedKeyHash, tokenURI_)` | `MINTER_ROLE` | Mints a new iNFT-7857. Emits `AgentMinted(tokenId, contentRoot, dimensions, owner)`. |

### Transfer & clone (ERC-7857)

| Function | Role | Effect |
|----------|------|--------|
| `transferWithSealedKey(from, to, tokenId, sealedKey, oracleProof)` | token owner / approved | Transfers + rotates sealed-key hash; emits `SealedKeyRotated`, `MetadataUpdated`. |
| `cloneAgent(tokenId, to, sealedKey, oracleProof)` *payable* | token owner / approved | Mints a child token sharing the parent's `contentRoot` with a fresh sealed key. Pays `cloneFeeWei` to `treasury`. Emits `AgentCloned`. |

### Authorization

| Function | Role | Effect |
|----------|------|--------|
| `authorizeUsage(tokenId, executor, permissions, expiresAt)` | token owner / approved | Grants permission bitmap to `executor` until `expiresAt`. Emits `UsageAuthorized(tokenId, executor, permissions, expiresAt, grantor)`. |
| `revokeUsage(tokenId, executor)` | token owner / approved | Clears the grant. Emits `UsageRevoked`. |
| `isUsageAuthorized(tokenId, executor) view` | — | Returns true if grant exists and not expired. |
| `getUsageGrant(tokenId, executor) view` | — | Returns full `UsageGrant` struct. |

### Burn

| Function | Role | Effect |
|----------|------|--------|
| `burn(tokenId)` | token owner / approved | Burns + clears payload + clears AP/BANKON bindings. `contentRoot` stays reserved (no re-mint). Emits `AgentBurned`. |

### Bindings

| Function | Role | Effect |
|----------|------|--------|
| `bindAgentId(tokenId, agentId)` | token owner / approved | One-shot mindX agent_id binding. Emits `AgentIdBound(tokenId, agentIdHash, agentId)`. |
| `offerOnAgenticPlace(tokenId, marketplace, price, isETH, paymentToken)` | token owner / approved | Records marketplace listing. Emits `AgenticPlaceListed`. |
| `bindBankonVault(tokenId, vault, vaultRef)` | token owner / approved | Records BANKON vault escrow. Emits `BankonVaultBound`. |

### Admin

| Function | Role | Effect |
|----------|------|--------|
| `pause()` / `unpause()` | `PAUSER_ROLE` | Emergency stop on writes (reads stay open). |
| `setOracle(newOracle)` | `DEFAULT_ADMIN_ROLE` | Atomically revokes old oracle's `ORACLE_ROLE` and grants the new one. Emits `OracleUpdated`. |
| `setTreasury(newTreasury)` | `TREASURER_ROLE` | Updates clone-fee recipient. |
| `setCloneFee(newFeeWei)` | `TREASURER_ROLE` | Updates per-clone fee. Emits `CloneFeeUpdated`. |
| `setDefaultRoyalty(receiver, feeBps)` | `DEFAULT_ADMIN_ROLE` | ERC-2981 default. Capped at 2500 bps (25%). |
| `setTokenRoyalty(tokenId, receiver, feeBps)` | `DEFAULT_ADMIN_ROLE` | Per-token override, same cap. |
| `sweepEth(amount)` | `TREASURER_ROLE` | Operator escape hatch for stuck ETH. |
| `grantRole / revokeRole` | `DEFAULT_ADMIN_ROLE` | Standard AccessControl. |

### Reads

`name()`, `symbol()`, `totalMinted()`, `exists(tokenId)`, `ownerOf(tokenId)`, `tokenURI(tokenId)`, `paused()`, `oracle()`, `treasury()`, `cloneFeeWei()`, `oracleNonce(tokenId)`, `isRootUsed(root)`, `getPayload(tokenId)`, `cloneCount(tokenId)`, `clonedFrom(tokenId)`, `getAgentId(tokenId)`, `getAgentIdHash(tokenId)`, `agenticPlaceFor(tokenId)`, `bankonVaultFor(tokenId)`, `bankonVaultRef(tokenId)`, `royaltyInfo(tokenId, salePrice)`, `validDimensions()`, `eip712Domain()`, `sealedKeyDigest(...)`, `cloneDigest(...)`, plus all `AccessControl` introspection.

## Interactive UI — `/inft7857`

[`mindx_backend_service/inft7857.html`](../mindx_backend_service/inft7857.html) is a single-file, dependency-light console covering every aspect of the contract. Loads ethers v6 from a CDN; uses MetaMask (or any injected wallet); auto-loads `openagents/deployments/galileo.json` to populate the contract address.

### Tabs

| Tab | What it does |
|-----|--------------|
| **Overview** | Reads name, symbol, totalMinted, paused, oracle, treasury, cloneFee, contract address. Shows the connected wallet's role grants (`DEFAULT_ADMIN_ROLE`, `MINTER_ROLE`, `ORACLE_ROLE`, `PAUSER_ROLE`, `TREASURER_ROLE`) with green-check / red-cross indicators. |
| **Mint** | Form-driven `mintAgent` call with all 11 dimensions in a dropdown, random sealed-key+hash generator, recipient + content-root + URI inputs, gas-cost preview. |
| **Inspect Token** | Single tokenId lookup that returns *every* on-chain field: payload, owner, agent_id, AP listing, BANKON vault, royalty receiver+amount (computed for any sale price), clone count, parent (if cloned), tokenURI, oracle nonce. Also has a sub-form to check `isUsageAuthorized` per (token, executor) pair. |
| **Transfer (sealed key)** | Calls `transferWithSealedKey`. **Demo-mode oracle signing**: paste an oracle private key, the UI fetches `eip712Domain()`, computes the digest via `sealedKeyDigest(...)`, signs locally, and pre-fills the proof input. Strong on-page warning to never paste mainnet keys. |
| **Clone** | Calls `cloneAgent`. Same demo-mode signing helper using `cloneDigest(...)`. Value field for the clone fee. |
| **Authorize / Revoke** | `authorizeUsage` form with permissions bitmap + +7-day / +30-day expiry helpers. Separate `revokeUsage` card. |
| **Burn** | `burn(tokenId)` with prominent irreversibility warning. |
| **Bind** | Three cards — `bindAgentId` (mindX agent identifier), `offerOnAgenticPlace` (price + payment-token + ETH-or-ERC20 toggle), `bindBankonVault` (vault address + opaque ref, with a "generate ref from agent_id" helper). |
| **Admin** | Pause/unpause, grant/revoke any role, setOracle, setTreasury, setCloneFee, setDefaultRoyalty, setTokenRoyalty, sweepEth — each gated behind the visually-indicated role. |

### Live event log

Subscribes to all 14 contract events (`AgentMinted`, `MetadataUpdated`, `SealedKeyRotated`, `UsageAuthorized`, `UsageRevoked`, `AgentCloned`, `AgentBurned`, `OracleUpdated`, `AgentIdBound`, `AgenticPlaceListed`, `BankonVaultBound`, `CloneFeeUpdated`, `TreasuryUpdated`, `Paused`/`Unpaused`) and renders them in a scrolling pane with humanized formatting. Shows a pulsing live-dot when a wallet is connected and the contract is loaded.

## Deployment

The contract deploys via `openagents/deploy/deploy_galileo.sh` to 0G Galileo testnet (chain 16602):

```bash
export ZEROG_PRIVATE_KEY=0x...
export ROYALTY_RECEIVER=0x...      # defaults to deployer
export ROYALTY_FEE_BPS=250         # 2.5%
./openagents/deploy/deploy_galileo.sh
# writes openagents/deployments/galileo.json
```

The contract's constructor wires:
- name + symbol
- DEFAULT_ADMIN_ROLE → admin
- royalty receiver + bps
- oracle (granted ORACLE_ROLE)
- treasury
- initial cloneFeeWei

After deploy, the [/inft7857](https://mindx.pythai.net/inft7857) UI auto-fetches the address from `openagents/deployments/galileo.json` (served by the `/openagents/deployments/{filename}` route mounted in `mindx_backend_service/main_service.py`).

The same script can target Sepolia, Arbitrum Sepolia, Base Sepolia, or any EVM chain by changing `ZEROG_RPC_URL` / `ZEROG_CHAIN_ID`.

## Integration paths

### mindX (agent infrastructure)
- `agents/core/id_manager_agent.py` mints a wallet for each new agent. The mintAgent call can be wired so that every agent is born with an iNFT-7857 representing its persona+memory bundle (the bundle uploaded to 0G Storage via `agents/storage/zerog_provider.py`).
- The `agentId` binding uses `bindAgentId(tokenId, agent_id)` immediately after mint — same string as the agent's mindX identifier, hashed for indexer queries.

### AgenticPlace (marketplace)
- AgenticPlace's contract is granted `MINTER_ROLE` so it can mint iNFTs on behalf of marketplace listings.
- When a user lists an iNFT for hire, the front-end calls `offerOnAgenticPlace(tokenId, marketplace, price, isETH, paymentToken)` to record the binding on chain. The marketplace's separate `offerSkill()` call (see [`THOT/marketplace/IAgenticPlace.sol`](../daio/contracts/THOT/marketplace/IAgenticPlace.sol)) handles the actual order book.

### BANKON (vault)
- The sealed AES key for an iNFT can be escrowed in a BANKON vault. The vault holds the key encrypted under the owner's BANKON identity; `bindBankonVault(tokenId, vault, vaultRef)` records the relation on chain. On owner-authenticated retrieval, the vault releases the key to the owner who can then decrypt the 0G Storage payload.
- BANKON's vault contract is a candidate for `MINTER_ROLE` so it can mint iNFTs as part of credential-bundle issuance.

## Security model — what's protected, what's NOT

**Protected on chain:**
- Standard ERC-721 transfers cannot bypass the sealed-key handoff (gating).
- Sealed-key handoffs cannot be replayed (per-token nonce + EIP-712 domain).
- Oracle proofs cannot be forged unless the oracle private key is compromised (ECDSA recovery against a static `oracle` address).
- Content roots cannot be silently re-minted after burn.
- Royalties are capped at 25%.
- Admin actions are role-gated, not owner-gated (multi-party authority possible).

**NOT protected on chain (must be enforced off chain):**
- The sealed-key ciphertext is delivered off chain. The contract only stores its hash. If the off-chain delivery channel is compromised, an attacker could intercept the ciphertext.
- The intelligence payload itself lives in 0G Storage / IPFS. Nothing on chain enforces decryption-correctness — the oracle attests that the ciphertext was generated for the new owner, but a malicious off-chain TEE could lie.
- `permissions` bitmap is interpreted off chain. The contract just stores it; agents using it must validate the bits themselves.

**Operator-action items before mainnet:**
1. Audit the ECDSA oracle. Ideal: ZK proof of TEE re-encryption rather than a single signer.
2. Use `Ownable2Step`-equivalent for the `DEFAULT_ADMIN_ROLE` holder — the current contract requires the admin to grant the new admin manually (two-tx pattern via `grantRole` + `renounceRole`).
3. Front-running: clone-fee changes between user broadcast and mining can cause `CloneFeeUnderpaid`. Add a `maxFee` param to `cloneAgent` if mainnet liveness matters.
4. The legacy `iNFT.sol` does not compile under the bundled OpenZeppelin v5 — preserved untouched for timestamp evidence; do not deploy it to mainnet.
